# 房间排序、结算状态与头像共享修复

## 变更级别

L2：涉及房间页展示规则、结算后的跨页面状态一致性、云存储头像共享路径。

## 变更目的

修复房间多人协同时的三个问题：

- 玩家入口排序与当前业务规则不一致。
- 结算后首页仍显示已结算房间。
- 头像保存为本机临时路径，其他用户设备无法显示。

## 规格说明

- 房间玩家入口展示顺序固定为：台面、茶水费、当前用户、其他玩家。
- 结算时先把 `status: 'settled'` 写入本地缓存，再等待云端保存完成后跳转结算页。
- 首页加载进行中房间时，以本地缓存中的同房间状态作为覆盖依据；本地已结算的房间不得继续显示在首页进行中列表。
- 用户选择头像后，如果头像是本机临时路径，需要上传到微信云存储，并把云文件 `fileID` 保存到用户资料或房间玩家资料。
- 已经是 `cloud://`、`http://`、`https://`、空值的头像地址不重复上传。

## 兼容策略

- 旧房间中已有本地临时头像路径时，不强制迁移；用户重新选择头像、创建房间或加入房间时会自动尝试转为云存储地址。
- 云存储上传失败时保留原头像路径或使用本地临时路径作为降级，不阻断创建、加入和记账主流程。
- 首页仍以云端 `rooms.status = 'playing'` 为主查询条件，同时用本地缓存剔除已结算房间。

## 验收点

- 房间页玩家入口顺序为：台面、茶水费、自己、其他玩家。
- 点击结算并返回首页后，刚结算的房间不再显示在进行中房间列表。
- A、B 两个用户选择头像后，进入同一房间能看到对方头像。
- 未配置或上传失败时，头像失败不影响创建房间、加入房间、支付、台面和结算。
