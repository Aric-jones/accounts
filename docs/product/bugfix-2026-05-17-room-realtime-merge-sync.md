# 房间实时合并同步修复

## 变更级别

L2：涉及房间实时同步策略、云函数写入契约、交易撤销与头像读取。

## 变更目的

修复多人房间中本地整房间覆盖云端造成的数据回退问题，并让茶水费、头像、交易记录在云端变更后及时刷新。

## 规格说明

- 房间页以云端 `rooms` 文档 watch 作为实时刷新主路径。
- watch 收到云端房间快照后直接刷新页面和本地缓存，不再用本地 `updatedAt` 阻挡云端新数据。
- 房间保存仍走 `recordScore` 云函数，但云函数不再简单整份覆盖：
  - `players` 按 `id` 合并，保留云端已有玩家。
  - `transactions` 按 `id` 合并，保留云端已有交易。
  - `deletedTransactionIds` 用于撤销时删除指定交易，避免合并时把撤销记录重新带回来。
  - 茶水费比例、收取模式、结算状态等字段按本次提交字段更新。
- 云函数保存成功后返回合并后的房间，小程序端立即用返回房间刷新本地缓存和页面。
- 云存储头像 `cloud://` 文件 ID 在展示前转换为临时 URL，避免部分 `<image>` 组件无法直接显示云文件 ID。

## 兼容策略

- 旧交易没有 `id` 时按 `timestamp/from/to/amount` 补充稳定合并键，避免重复。
- 保存失败时仍保留本地缓存，但页面会提示云端保存失败。
- 头像临时 URL 只用于页面展示，不写回房间主数据，避免过期 URL 污染云端数据。

## 验收点

- A、B 同时在房间内操作，任一方新增交易后，另一方无需重新进房即可看到。
- A 离开数轮后再操作，不会把 B 期间新增交易覆盖掉。
- 撤销只删除本次指定交易，不会因为合并逻辑重新出现。
- 茶水费设置保存后，其他用户房间页及时显示新比例和模式。
- 双方头像使用云存储文件 ID 保存时，房间页能显示对方头像。
