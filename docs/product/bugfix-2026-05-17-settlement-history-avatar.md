# 结算状态、个人历史与头像展示修复

## 变更级别

L2：涉及房间实时状态、历史记录归属、云函数删除契约、头像展示解析。

## 变更目的

修复结算后其他用户仍可继续记账、历史记录删除影响所有人、以及云存储头像在其他设备显示为纯色块的问题。

## 规格说明

- 房间云端状态变为 `settled` 后，仍停留在房间页的用户必须提示房间已结束，并跳转到结算页。
- 房间页所有记账入口在执行前检查 `room.status`，已结算房间不允许继续支付、台面、茶水费、撤销或编辑房间玩家资料。
- 历史记录按用户维度隐藏：删除历史只写入当前用户的隐藏列表，不删除 `rooms` 原始房间文档。
- 清空历史只影响当前用户，不影响其他参与者查看同一局。
- 结算后每个参与者通过云端 `rooms` 读取同一份结算备份；本地可移除进行中入口，但历史备份仍可从云端获取。
- 头像展示必须先把 `cloud://` 文件 ID 转为临时 URL；未拿到临时 URL 时回退显示昵称首字，不直接渲染 `cloud://`。
- 历史页和结算页也使用相同头像解析逻辑。

## 兼容策略

- 旧历史删除如果已误删云端房间，无法自动恢复；新版本开始不再物理删除房间。
- 没有头像或头像临时 URL 解析失败时，统一显示昵称首字。
- `getHistory` 兼容旧 `history` 查询，同时过滤当前用户隐藏过的房间。

## 验收点

- A 结算后，B 的房间页自动提示结束并跳转结算页。
- 房间已结算后，B 不能继续支付或修改计数。
- A 删除历史后，B 的历史列表仍能看到该房间。
- 设置头像的玩家在其他用户设备上显示头像；未设置头像显示昵称首字。
