# 首页删除房间云函数错误提示修复

## 变更级别

L2：涉及首页删除交互和 `getHistory` 云函数契约诊断。

## 问题

首页删除自己创建的房间时报：

```text
delete owned room failed Error: delete failed
```

## 根因

前端调用 `getHistory(action: 'deleteOwnedRoom')` 后，如果云端仍是旧版 `getHistory`，旧函数不认识该 action，可能返回空结果。前端此前只给默认错误 `delete failed`，无法判断是“权限失败”“房间不存在”还是“云函数未部署新版”。

## 修复规格

- 前端必须区分：
  - 云函数未部署新版或未返回标准 `{ code }`。
  - 当前用户不是创建者。
  - 房间已不存在。
  - 其他云端删除失败。
- `getHistory` 对未知 action 返回明确错误，不再静默返回空。
- 删除失败时弹出可操作提示，而不是只显示“删除失败”。

## 验收点

- [ ] 未部署新版 `getHistory` 时，删除失败提示“请部署 getHistory 云函数”。
- [ ] 非创建者删除时，提示只有房主可删除。
- [ ] 房间已不存在时，本地引用可清理，首页不再显示该房间。
- [ ] 部署新版云函数后，创建者删除成功。
