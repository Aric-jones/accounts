## 1. 小程序基础概念

### 1.1 什么是微信小程序

微信小程序是一种**不需要下载安装**即可使用的应用，运行在微信客户端内部。它有自己的一套开发框架，包括：

- **渲染层**：使用 WXML（类似 HTML）+ WXSS（类似 CSS）
- **逻辑层**：使用 JavaScript（运行在 JSCore 中，不是浏览器环境）
- **原生组件**：如 `<map>`、`<camera>`、`<canvas>` 由微信原生渲染

### 1.2 架构模型

```
┌─────────────────────────────────────┐
│           微信客户端                  │
│  ┌──────────┐    ┌──────────────┐   │
│  │  渲染层   │    │   逻辑层      │   │
│  │  WXML    │◄──►│  JavaScript  │   │
│  │  WXSS    │    │  (JSCore)    │   │
│  │  (WebView)│    │              │   │
│  └──────────┘    └──────────────┘   │
│         ▲               ▲           │
│         └───── 微信原生 ──┘           │
└─────────────────────────────────────┘
```

渲染层和逻辑层分离，通过微信原生层做数据传递（`setData`）。这意味着：

- **没有 DOM 操作**，不能用 `document.getElementById`
- **没有 BOM**，不能用 `window`、`location`
- 所有视图更新必须通过 `this.setData()` 驱动

### 1.3 与 H5/App 的区别

| 特性 | 小程序 | H5 | 原生 App |
|------|--------|-----|---------|
| 安装 | 无需安装 | 无需安装 | 需下载 |
| 体积 | ≤20MB（分包后） | 无限制 | 无限制 |
| 性能 | 接近原生 | 较差 | 最优 |
| API | 微信封装 | 浏览器API | 系统API |
| 审核 | 需审核 | 无 | 需审核 |
| 获客 | 微信生态 | 搜索引擎 | 应用商店 |

## 2. 开发环境搭建

### 2.1 注册小程序账号

1. 访问 [mp.weixin.qq.com](https://mp.weixin.qq.com)
2. 点击「立即注册」→ 选择「小程序」
3. 填写邮箱、密码、主体信息
4. **个人主体**：只需身份证 + 微信扫码验证
5. 注册成功后获得 **AppID**（如 `wx4901bd2a62b4636e`）

> 个人主体限制：不能使用微信支付、不能开通会员、部分类目不可选

### 2.2 安装微信开发者工具

下载地址：[developers.weixin.qq.com/miniprogram/dev/devtools/download.html](https://developers.weixin.qq.com/miniprogram/dev/devtools/download.html)

- 支持 Windows / macOS
- 集成了代码编辑器、模拟器、调试器
- 可以直接预览、真机调试、上传代码

### 2.3 创建项目

打开开发者工具 → 新建项目：

- **项目名称**：自定义
- **目录**：选择一个空文件夹
- **AppID**：填写你的真实 AppID
- **模板**：选「不使用模板」或「云开发」
- **语言**：JavaScript（推荐）或 TypeScript

## 4. WXML 模板语法

WXML（WeiXin Markup Language）是小程序的视图模板语言，类似 HTML 但有自己的标签和语法。

### 4.1 基础标签

| WXML 标签 | 类似 HTML | 用途 |
|-----------|----------|------|
| `<view>` | `<div>` | 通用容器 |
| `<text>` | `<span>` | 文本 |
| `<image>` | `<img>` | 图片 |
| `<scroll-view>` | 无 | 可滚动容器 |
| `<button>` | `<button>` | 按钮 |
| `<input>` | `<input>` | 输入框 |
| `<navigator>` | `<a>` | 导航链接 |
| `<swiper>` | 无 | 轮播图 |

### 4.2 数据绑定

```xml
<!-- 文本绑定 -->
<text>{{message}}</text>

<!-- 属性绑定 -->
<view class="box-{{type}}">内容</view>

<!-- 布尔属性 -->
<button disabled="{{isDisabled}}">按钮</button>

<!-- 三元表达式 -->
<text>{{score > 0 ? '+' : ''}}{{score}}</text>

<!-- 算术运算 -->
<text>总价: {{price * quantity}}</text>
```

**注意：** WXML 中不能调用 JS 方法（如 `.toFixed()`、`.map()`），复杂逻辑必须在 JS 中预处理后传入。

### 4.3 条件渲染

```xml
<!-- wx:if / wx:elif / wx:else -->
<view wx:if="{{status === 'loading'}}">加载中...</view>
<view wx:elif="{{status === 'empty'}}">暂无数据</view>
<view wx:else>{{content}}</view>

<!-- hidden（不销毁DOM，只是隐藏） -->
<view hidden="{{!show}}">隐藏内容</view>
```

| 方式 | 特点 | 适用场景 |
|------|------|---------|
| `wx:if` | 惰性渲染，条件为 false 不渲染 | 切换不频繁 |
| `hidden` | 始终渲染，CSS 控制显隐 | 切换频繁 |

### 4.4 列表渲染

```xml
<!-- 基本用法 -->
<view wx:for="{{list}}" wx:key="id">
  {{index}}: {{item.name}}
</view>

<!-- 自定义变量名 -->
<view wx:for="{{players}}" wx:for-item="player" wx:for-index="idx" wx:key="id">
  第{{idx + 1}}名: {{player.nickname}}
</view>

<!-- 嵌套循环 -->
<view wx:for="{{rounds}}" wx:key="roundNum">
  <view wx:for="{{item.scores}}" wx:for-item="score" wx:key="*this">
    {{score}}
  </view>
</view>
```

**`wx:key` 很重要**：提供唯一标识，帮助框架高效更新列表。值可以是：
- 字符串：代表 item 的某个属性名（如 `"id"`）
- `*this`：item 本身是唯一字符串/数字

### 4.5 事件绑定

```xml
<!-- 冒泡事件 -->
<button bindtap="onTap">点击</button>

<!-- 非冒泡事件 -->
<button catchtap="onTap">点击（阻止冒泡）</button>

<!-- 传参（通过 data- 属性） -->
<view bindtap="onSelect" data-id="{{item.id}}" data-name="{{item.name}}">
  {{item.name}}
</view>
```

JS 中接收参数：

```javascript
onSelect(e) {
  const { id, name } = e.currentTarget.dataset
  console.log(id, name)
}
```

### 4.6 模板与引用

```xml
<!-- 定义模板 -->
<template name="playerCard">
  <view class="card">
    <text>{{nickname}}</text>
    <text>{{score}}</text>
  </view>
</template>

<!-- 使用模板 -->
<template is="playerCard" data="{{...item}}" />

<!-- 引入外部文件 -->
<import src="templates/card.wxml" />
<include src="common/header.wxml" />
```

### 4.7 WXS（WeiXin Script）

WXS 是在 WXML 中使用的脚本语言，用于处理模板中的数据格式化：

```xml
<wxs module="fmt">
  module.exports.formatScore = function(score) {
    return score > 0 ? '+' + score : '' + score
  }
</wxs>

<text>{{fmt.formatScore(item.score)}}</text>
```

> WXS 的运行性能比在 JS 中通过 setData 传递数据更高，适合频繁的数据格式化。

## 5. WXSS 样式语法

### 5.1 基本规则

WXSS 兼容大部分 CSS 语法，但有以下不同：

- **尺寸单位：rpx**（responsive pixel）
  - 750rpx = 屏幕宽度（不管什么设备）
  - iPhone 6/7/8：1rpx = 0.5px
  - 推荐统一使用 rpx
- **不支持**：`*` 通配选择器、一些 CSS3 属性
- **支持**：`@import` 导入外部样式

```css
/* 尺寸示例 */
.card {
  width: 690rpx;      /* 接近全屏宽 */
  margin: 0 30rpx;    /* 左右各留30rpx */
  padding: 24rpx;
  border-radius: 16rpx;
  font-size: 28rpx;   /* 约14px */
}
```

### 5.2 rpx 与 px 对照表

| rpx | 在 iPhone 6 (375px) 上 |
|-----|----------------------|
| 1rpx | 0.5px |
| 24rpx | 12px |
| 28rpx | 14px |
| 32rpx | 16px |
| 750rpx | 375px（满屏宽） |

### 5.3 样式导入

```css
/* app.wxss */
@import "styles/variables.wxss";
@import "styles/common.wxss";
```

### 5.4 CSS 变量（自定义属性）

```css
/* app.wxss 定义全局变量 */
page {
  --color-primary: #1A6B4A;
  --color-bg: #FAFAF5;
  --color-text: #2D2D2D;
  --color-card: #FFFFFF;
}

/* 使用 */
.card {
  background: var(--color-card);
  color: var(--color-text);
}
```

### 5.5 Flex 布局（最常用）

```css
/* 水平居中 */
.center {
  display: flex;
  align-items: center;
  justify-content: center;
}

/* 两端对齐 */
.between {
  display: flex;
  justify-content: space-between;
  align-items: center;
}

/* 横向滚动 */
.scroll-row {
  display: flex;
  white-space: nowrap;
  overflow-x: auto;
}
```

### 5.6 安全区域适配

```css
/* 适配iPhone X及以上底部安全区 */
.bottom-bar {
  padding-bottom: calc(16rpx + env(safe-area-inset-bottom));
}
```

## 6. JavaScript 逻辑层

### 6.1 App 全局实例

```javascript
// app.js
App({
  onLaunch() {
    // 小程序初始化时触发（全局只触发一次）
  },

  onShow() {
    // 小程序从后台切到前台
  },

  onHide() {
    // 小程序从前台切到后台
  },

  globalData: {
    userInfo: null,
    theme: 'light'
  }
})
```

任何页面中通过 `getApp()` 获取全局实例：

```javascript
const app = getApp()
console.log(app.globalData.userInfo)
```

### 6.2 Page 页面实例

```javascript
Page({
  data: {
    list: [],
    loading: false
  },

  onLoad(options) {
    // 页面加载，options 是 URL 参数
    // 如 /pages/room/room?id=xxx → options.id === 'xxx'
  },

  onShow() {
    // 页面显示（每次可见时触发）
  },

  onReady() {
    // 页面初次渲染完成
  },

  onHide() {
    // 页面隐藏（navigateTo 到其他页面时）
  },

  onUnload() {
    // 页面卸载（navigateBack 时）
  },

  // 自定义方法
  loadData() {
    this.setData({ loading: true })
    // ...
  }
})
```

### 6.3 setData — 核心中的核心

```javascript
// 基本用法
this.setData({ message: 'hello' })

// 修改数组某一项
this.setData({ 'list[0].name': '新名字' })

// 修改对象某个属性
this.setData({ 'userInfo.nickname': '张三' })

// 回调（数据更新且渲染完成后）
this.setData({ count: 10 }, () => {
  console.log('渲染完成')
})
```

**性能规则：**
- `setData` 会将数据从逻辑层传到渲染层，数据量越大越慢
- 只传变化的数据，不要每次传整个 data
- 避免频繁调用 setData（如 scroll 事件中）
- 不要在 data 中存不需要渲染的数据

```javascript
// ❌ 错误：每次传整个列表
this.setData({ list: this.data.list })

// ✅ 正确：只更新变化的项
this.setData({ [`list[${index}].score`]: newScore })
```

### 6.4 模块化

```javascript
// utils/util.js
const formatTime = (date) => {
  const h = date.getHours().toString().padStart(2, '0')
  const m = date.getMinutes().toString().padStart(2, '0')
  return `${h}:${m}`
}

module.exports = { formatTime }

// 页面中使用
const { formatTime } = require('../../utils/util')
```

> 小程序原生不支持 ES Modules（`import/export`），使用 CommonJS（`require/module.exports`）。开启增强编译后可使用 `async/await`。

## 7. 页面生命周期

```
                    ┌──────────┐
                    │  onLoad  │ ← 页面创建（只执行一次）
                    └────┬─────┘
                         ↓
                    ┌──────────┐
                    │  onShow  │ ← 页面显示（每次可见）
                    └────┬─────┘
                         ↓
                    ┌──────────┐
                    │ onReady  │ ← 首次渲染完成（只一次）
                    └────┬─────┘
                         ↓
               ┌─────────┴─────────┐
               ↓                   ↓
         ┌──────────┐        ┌──────────┐
         │  onHide  │        │ onUnload │
         │(切后台/   │        │(返回上页/ │
         │ navigateTo)│       │ redirectTo)│
         └────┬─────┘        └──────────┘
              ↓
         ┌──────────┐
         │  onShow  │ ← 重新可见
         └──────────┘
```

### 各阶段典型用法

| 生命周期 | 用途 |
|---------|------|
| `onLoad(options)` | 获取 URL 参数、初始化数据、发起首次请求 |
| `onShow` | 刷新数据（如从其他页面返回时更新） |
| `onReady` | 获取组件实例、操作 canvas |
| `onHide` | 清除定时器、暂停音视频 |
| `onUnload` | 清理资源 |

### 8.2 常用 API

```javascript
// === 路由导航 ===
wx.navigateTo({ url: '/pages/room/room?id=123' })  // 保留当前页，跳转
wx.redirectTo({ url: '/pages/room/room?id=123' })   // 关闭当前页，跳转
wx.navigateBack({ delta: 1 })                        // 返回上一页
wx.switchTab({ url: '/pages/index/index' })           // 切换TabBar页
wx.reLaunch({ url: '/pages/index/index' })            // 关闭所有页重新打开

// === 弹窗交互 ===
wx.showToast({ title: '成功', icon: 'success' })
wx.showModal({
  title: '确认',
  content: '确定删除吗？',
  success(res) { if (res.confirm) { /* 用户点了确定 */ } }
})
wx.showActionSheet({
  itemList: ['选项A', '选项B'],
  success(res) { console.log(res.tapIndex) }
})

// === 网络请求 ===
wx.request({
  url: 'https://api.example.com/data',
  method: 'POST',
  data: { key: 'value' },
  header: { 'Authorization': 'Bearer xxx' },
  success(res) { console.log(res.data) },
  fail(err) { console.error(err) }
})

// === 用户信息 ===
wx.getUserProfile({
  desc: '用于完善用户资料',
  success(res) { console.log(res.userInfo) }
})

// === 分享 ===
// 在 Page 中定义
onShareAppMessage() {
  return {
    title: '分享标题',
    path: '/pages/index/index?from=share',
    imageUrl: '/images/share.png'
  }
}

// === 扫码 ===
wx.scanCode({
  success(res) { console.log(res.result) }
})

// === 剪贴板 ===
wx.setClipboardData({ data: '复制的内容' })

// === 震动反馈 ===
wx.vibrateShort({ type: 'medium' })
```

### 8.3 自定义组件

创建 `components/player-card/player-card`：

```javascript
// player-card.js
Component({
  properties: {
    name: { type: String, value: '' },
    score: { type: Number, value: 0 }
  },

  data: {
    displayScore: ''
  },

  observers: {
    'score': function(val) {
      this.setData({ displayScore: val > 0 ? '+' + val : '' + val })
    }
  },

  methods: {
    onTap() {
      this.triggerEvent('tap', { name: this.data.name })
    }
  }
})
```

```xml
<!-- player-card.wxml -->
<view class="card" bindtap="onTap">
  <text>{{name}}</text>
  <text>{{displayScore}}</text>
</view>
```

使用时在页面 .json 中注册：


## 9. 数据存储方案

### 9.1 本地存储（Storage）

```javascript
// 同步写入（推荐简单场景）
wx.setStorageSync('key', { name: '张三', score: 100 })

// 同步读取
const data = wx.getStorageSync('key')

// 异步写入
wx.setStorage({ key: 'key', data: value, success() {} })

// 删除
wx.removeStorageSync('key')

// 清空所有
wx.clearStorageSync()

// 查看存储信息
const info = wx.getStorageInfoSync()
console.log(info.currentSize)  // 当前已用 KB
console.log(info.limitSize)    // 上限 KB
```

**限制：**
- 单个 key 上限 1MB
- 总上限 10MB
- 同步操作会阻塞，大数据用异步
- 数据持久化在用户设备上，清除小程序缓存则丢失

**适用场景**：用户设置、游戏记录、草稿数据、缓存

### 9.2 云数据库

微信云开发提供 **JSON 文档型数据库**（类似 MongoDB）：

```javascript
// 获取数据库引用
const db = wx.cloud.database()
const rooms = db.collection('rooms')

// 增
const res = await rooms.add({ data: { name: '牌局', players: [] } })
console.log(res._id)  // 自动生成的文档ID

// 查
const res = await rooms.doc('文档ID').get()
console.log(res.data)

// 查多条
const res = await rooms.where({ status: 'playing' })
  .orderBy('createdAt', 'desc')
  .limit(20)
  .get()

// 改
await rooms.doc('文档ID').update({
  data: { status: 'settled' }
})

// 删
await rooms.doc('文档ID').remove()
```

**权限规则**：
- 默认：用户只能读写自己创建的数据（`_openid` 字段匹配）
- 可在云开发控制台修改集合的安全规则
- 云函数中操作数据库无权限限制

### 9.3 存储方案选型

| 方案 | 容量 | 多端同步 | 成本 | 适用场景 |
|------|------|---------|------|---------|
| 本地存储 | 10MB | ❌ | 免费 | 单机应用、缓存、草稿 |
| 云数据库 | 2GB起 | ✅ | 有免费额度 | 多人协作、数据持久化 |
| 自建服务器 | 无限 | ✅ | 按服务器付费 | 大规模应用 |

## 11. 网络请求

### 11.1 wx.request

```javascript
wx.request({
  url: 'https://api.example.com/users',
  method: 'GET',
  header: {
    'Content-Type': 'application/json',
    'Authorization': 'Bearer ' + token
  },
  success(res) {
    if (res.statusCode === 200) {
      console.log(res.data)
    }
  },
  fail(err) {
    console.error('网络错误', err)
  }
})
```

### 11.2 封装 Promise

```javascript
// utils/request.js
const request = (url, method = 'GET', data = {}) => {
  return new Promise((resolve, reject) => {
    wx.request({
      url: 'https://api.example.com' + url,
      method,
      data,
      header: {
        'Authorization': 'Bearer ' + wx.getStorageSync('token')
      },
      success(res) {
        if (res.statusCode >= 200 && res.statusCode < 300) {
          resolve(res.data)
        } else {
          reject(res)
        }
      },
      fail: reject
    })
  })
}

module.exports = { request }
```

## 14. 部署与上线

### 14.1 上传代码

1. 开发者工具 → 「上传」按钮
2. 填写版本号（如 `1.0.0`）和项目备注
3. 上传成功后代码进入「开发版本」

### 14.2 提交审核

1. 登录 mp.weixin.qq.com
2. 「管理」→「版本管理」
3. 在「开发版本」列找到刚上传的版本 → 「提交审核」
4. 填写审核信息：
   - 版本描述
   - 服务类目（如「工具 → 信息查询」）
   - 测试账号（如果需要登录）
   - 功能页面截图

### 14.3 审核要点

**常见被拒原因：**

| 原因 | 解决方案 |
|------|---------|
| 功能不完整 | 确保所有页面可用，无空白页 |
| 涉及资质 | 棋牌类需要「网络文化经营许可证」，记账工具类一般不需要 |
| 诱导分享 | 不能强制分享才能使用功能 |
| 虚拟支付 | 个人主体不能有虚拟支付，不要出现 ¥ 符号的购买按钮 |
| 内容违规 | 不能有赌博、色情等内容 |
| 广告过多 | 广告不能遮挡核心功能 |

**记账类小程序建议类目**：`工具 → 信息查询` 或 `生活服务 → 生活记录`

### 14.4 审核时间

- 首次提审：1-7 天
- 后续更新：通常 1-2 天，快的几小时
- 加急通道：每年有一定次数的加急机会（2小时内）

### 14.5 发布上线

审核通过后 → 在版本管理页面点「发布」→ 全量发布 or 灰度发布

### 14.6 版本管理策略

```
开发版本（上传后）
    ↓ 提交审核
审核版本（审核中）
    ↓ 审核通过
线上版本（对外可用）
```

建议使用语义化版本号：
- `1.0.0` → 首次发布
- `1.0.1` → Bug 修复
- `1.1.0` → 新功能
- `2.0.0` → 重大改版

## 16. 常见坑与解决方案

### 16.1 setData 的坑

```javascript
// ❌ setData 传入 undefined 会导致错误
this.setData({ value: undefined })  // 报错！

// ✅ 用 null 或 '' 代替
this.setData({ value: null })

// ❌ 直接修改 data（不会触发视图更新）
this.data.list.push(item)

// ✅ 必须通过 setData
const list = this.data.list
list.push(item)
this.setData({ list })
```

### 16.2 WXML 的坑

```xml
<!-- ❌ WXML 不支持调用方法 -->
<text>{{score.toFixed(2)}}</text>

<!-- ✅ 在 JS 中预计算 -->
<!-- JS: this.setData({ scoreText: score.toFixed(2) }) -->
<text>{{scoreText}}</text>

<!-- ❌ 不支持 ES6 模板字符串 -->
<text>{{'Hello ' + name}}</text>  <!-- 这样也不行 -->

<!-- ✅ 在 JS 中拼接好再传入 -->
```

### 16.3 页面栈限制

小程序页面栈最多 10 层：
- `navigateTo` 会压入新页面（栈 +1）
- `navigateBack` 弹出页面（栈 -1）
- `redirectTo` 替换当前页面（栈不变）
- `reLaunch` 清空所有页面重新打开

> 如果频繁 navigateTo 不返回，会导致页面栈溢出无法跳转。

### 16.4 插件使用

```json
// app.json 中声明插件
{
  "plugins": {
    "WechatSI": {
      "version": "0.3.6",
      "provider": "wx069ba97219f66d99"
    }
  }
}
```

**必须先在微信公众平台后台添加插件授权**，否则小程序会启动失败。

### 16.5 包体积超限

| 区域 | 限制 |
|------|------|
| 主包 | 2MB |
| 单个分包 | 2MB |
| 总体积 | 20MB |

解决方案：
- 图片放 CDN 或云存储，不放本地
- 使用分包加载
- 压缩代码（开发者工具自动压缩）
- 删除未使用的文件

### 16.6 真机与模拟器差异

| 差异点 | 模拟器 | 真机 |
|--------|--------|------|
| 网络 | 无域名校验可关 | 严格校验 HTTPS |
| 性能 | 较快 | 低端机可能卡顿 |
| API | 部分 API 不可用 | 完整支持 |
| 样式 | Chrome 渲染 | 微信内核渲染 |
